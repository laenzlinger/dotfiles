# Theming

Base16 colors managed by [tinty](https://github.com/tinted-theming/tinty).
Dark/light switching handled by [darkman](https://darkman.whynothugo.nl/).

## How it works

1. Darkman detects dark/light mode and runs `~/.local/share/darkman/tinty.sh`
2. Tinty applies a base16 scheme and runs hooks for each configured app
3. Each hook either copies a theme file or runs a `generate-colors.sh` script

## Color flow

```
tinty apply <scheme>
  → waybar: colors.css (SIGUSR2 reload)
  → sway: base16.config (applied on next sway reload)
  → wezterm: tinted-theming.toml
  → fzf: sourced in shell
  → btop: colors.theme (SIGUSR2 reload)
  → rofi: generate-colors.sh (derives from waybar colors.css)
  → wob: generate-colors.sh (derives from waybar colors.css)
  → swaync: generate-colors.sh (derives from waybar colors.css)
  → obsidian: generate-colors.sh (derives from waybar colors.css, all vaults)
  → vivaldi: generate-theme.sh (live via CDP, or Preferences file fallback)
```

## Adding a new app

1. Create `~/.config/<app>/generate-colors.sh` that reads `~/.config/waybar/colors.css`
2. Add a `[[items]]` entry to `dot_config/tinted-theming/tinty/config.toml.tmpl`
3. Run `tinty install` to fetch any new repos
4. Test with `tinty apply <scheme>`

## Obsidian

Theme is generated by `~/.config/obsidian/generate-colors.sh`:
- Reads base16 colors from waybar's `colors.css`
- Applies to all vaults listed in `~/.config/obsidian/obsidian.json`
- Sets both `.theme-dark` and `.theme-light` to the same colors (tinty switches the scheme)
- Uses MesloLGS Nerd Font for monospace

## Vivaldi

Theme is generated by `~/.config/vivaldi/generate-theme.sh`:
- Reads base16 colors from waybar's `colors.css`
- Detects dark/light from base00 luminance
- Creates user themes "Base16 Dark" / "Base16 Light" in Vivaldi

Color mapping:
- `colorBg` ← base00 (main background)
- `colorWindowBg` ← base01 (toolbar background)
- `colorAccentBg` ← base01 (tab bar — subtle, close to background)
- `colorFg` ← base05 (text)
- `colorHighlightBg` ← base0D (selections, active elements)

### Live updates via CDP

Vivaldi is launched with `--remote-debugging-port=9222` (set in `~/.config/vivaldi-stable.conf`).
The generate script uses the Chrome DevTools Protocol to call `vivaldi.prefs.set()` directly,
which triggers an immediate UI redraw. If CDP is unavailable (Vivaldi not running), it falls
back to editing the `Preferences` JSON file.

Vivaldi's built-in "system" schedule mode does not react to the freedesktop
`org.freedesktop.appearance` portal on Wayland, so the script sets `vivaldi.themes.current`
directly and disables the schedule.

## Sway border colors

Sway base16 config is updated on theme switch but `swaymsg reload` is not triggered
(it causes a ~30s waybar restart). Border colors update on next manual reload (`Mod+Shift+c`).

## Dark/light switching

Darkman maps modes to schemes in `~/.local/share/darkman/tinty.sh`:
- dark → `base16-darktooth`
- light → `base16-gruvbox-light-medium`
